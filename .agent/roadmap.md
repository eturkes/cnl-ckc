### Decisions (plan-fixed)

- Query = exactly one yes/no `question/1`, final root condition; wh `query/2` recognized → hard error (M4 owns answer binding). A declarative goal is indistinguishable from a fact — rejected as query encoding.
- Copula normalization (versioned in spec): `object(X,C,countable,na,eq,1)` + `predicate(_,be,named(N),X)` in the same factual DRS → class fact `C(named(N))` carrying both token anchors; every other `be`/`object` arrangement = hard error.
- Event referents: erase iff declared in the same DRS domain and used exactly once; any reuse = hard error (lossless-erasure proof, not silent drop).
- IR v1 = single ground record: `cnl_ir_record(1).`, document line byte-verbatim from the M2 record, then facts/rules/queries with `fact_id/rule_id/query_id(sentence(S),clause(C))`, `var(N)` data (never native vars), `source(sentence(S),tokens([...]))` provenance; shape-fixed name-open predicates, sort vocabulary `{entity}`.
- Semantics: positive function-free Datalog, least Herbrand model; outcomes `proved`/`not_proved` (rendered unknown, never false; no false constructor in v1); NAF reserved in grammar + validator-rejected; stratification stated, trivially satisfied; safety = every var covered by positive body, facts/queries ground; positive dependency cycles rejected.
- Unsupported constructs = deterministic hard errors (no residuals in v1); `ir_tool_error(Stage,Class,Detail).` stderr line, adapter-style exits 0/1/2, zero stdout on failure, canonical fixed-point serialization (drs_canon writer options).
- Tooling: one `src/prolog/ir_tool.pl` CLI — subcommands `lower|validate|compile|run`, fresh `swipl -q -f none -F none` process per stage; semantic modules `drs_to_ir.pl`, `ir_validate.pl`, `ir_to_prolog.pl`, `inference_kernel.pl`, `explanation.pl`; reuse `drs_canon`; M2 modules untouched.
- Program artifact = directive-free versioned term stream, whitelist-validated before install into a private module; kernel = deterministic meta-interpreter, first proof under program clause order = canonical witness, replayed before emission; result record `cnl_answer_record(1).` + verbatim document line + answer/derivation certificate.
- Artifacts per docid: `.ir.pl`/`.program.pl`/`.result.pl`; fixtures `tests/fixtures/ir/` (hand-authored validator green/red inputs) + `tests/fixtures/slice/` (ACE/ulex docs + regenerated chain goldens); zero changes to `tools/ace_front_end.emm`/generated Python; orchestration = bash harness (M5 owns the real E-- CLI).
- Deferrals (named, M4+): intervals/rationals, temporal/dose algebra, direction/strength/certainty, explicit negation + false outcome, NAF execution/exceptions, recursion/tabling, conflict detection, proof enumeration/DAG sharing, prose rendering, program-digest binding inside answer records, wh answers, multi-document composition.

### Decisions (plan-fixed, D1–D10 vetted)

- **Negation (D1)**: v2's sole admitted negation = native `~/1` lowered to `naf(pred(Name,Args))`, admitted only as a rule-antecedent condition wrapping exactly one intransitive `predicate/3` (P13) or one copula pair (P15) whose entity is bound by the enclosing antecedent; the `~` sub-DRS must declare exactly its own consumed referents (event, or copula object+event). All classical `-/1` anywhere, root/consequent/question `~`, nested `~`/`-`, and any other `~` content = hard errors, new lower class `negation` (distinct from `unsupported`; probe-pinned). Docs state honestly: DRS report labels `~` NAF without fixing program semantics; v2 interprets it in its finite acyclic profile where perfect = well-founded = unique stable model coincide (Apt–Bol Thm 6.20 + Cors 7.6–7.7 p32 — numbering per the open PDFs, twice PDF-verified; a lens-reported "7.8–7.9" was wrong; M4.1 re-verifies section numbers against the PDF it cites), so the reading is semantics-robust; `not provably P` = no derivation under the closed program, never classical falsity.
- **Stratification (D2)**: dependency graph gains negative edges (rule head → each naf-target key); v2 rejects ANY cycle (positive or negative edge) — stronger than stratification (which permits positive-only cycles: AHV Prop 15.2.7; Krötzsch OASIcs 138 §6 Def.31 pp.7:21–7:22 = modern statement incl. SCC criterion), so a stratification exists trivially; docs claim only sufficiency, never necessity. Terminology guards for M4.1 docs text: state finiteness explicitly (finite function-free predicate graph — on infinite ground graphs no-negative-cycle is insufficient, well-foundedness required, Blair–Marek–Schlipf Lem.4.1); avoid the term-of-art "acyclic program" (Apt–Bezem = ground-level mapping subclass, ≠ predicate-dependency DAG) — say "cycle-free predicate dependency graph"; forbidden = any cycle containing ≥1 negative edge (no parity rule), subsumed by full acyclicity. Model naming: with naf present there is generally NO global least model (v1's "least Herbrand model" wording must not survive into v2 text; e.g. `p :- naf q` has incomparable minimal models {p},{q}) — call it the standard/stratified model, minimal + supported, per-stratum least fixpoint with lower strata frozen as EDB (ABW pp.107–112 Thms 7/8/11; AHV §15.2 Thm 15.2.10 + Prop 15.2.11 "minimal, not necessarily least"; Green et al. FnT-DB §2.3.2 "treat negated predicates as EDBs"), then cite the coincidences (= unique perfect = unique stable = total WFM: Apt–Bol Thms 6.10/6.20 + Cor 7.7; VGRS JACM 38(3) Thm 6.1). Class `cycle` widens to signed edges; recursion stays deferred. Kernel: stratum(p) = max(positive deps' strata, 1 + naf deps' strata) over the acyclic graph; evaluate strata ascending, existing v1 snapshot-pass fixpoint restricted to each stratum's clauses; naf test = ground atom absent from the store (lower strata complete for naf targets; positive same-stratum deps completed by the stratum's own fixpoint). Empty naf-target predicate = vacuously underivable, naf succeeds — sound under CWA (van Emden–Kowalski least model; Ferraris–Lee–Lifschitz trivial predicates); typo defense = ulex fail-closed + closed record universe, no declaration constructor (D10 resolved: labeled exception IDs + explicit closed-world declarations deferred with direction/strength to M5+).
- **NAF safety + position (D3)**: validator-enforced canonical body order = all positive literals, then all naf literals (source order within each block; lowering emits this order; interleaved antecedents rejected); every naf-literal variable must occur in a positive literal of the same rule (class `safety`); kernel asserts groundness of each substituted naf atom before testing (violation = internal `uncaught`). One body order serves bytes, `var(N)` numbering, execution, witnesses, certificate children.
- **Certificates (D4)**: naf body literal → ground leaf child `naf(Atom)` in the proof tree, positionally matching the cited clause; replay checks shape/groundness/substitution correspondence AND absence of `Atom` from the completed kernel store (store passed into replay; model-membership invariant, not just syntax). Trust boundary documented: saturation = kernel; absence-check = against that store; fresh-process determinism = reproducibility; external verification = recompute from the program. No model section in answer records.
- **Wh answers (D5)**: admit exactly the P01/P10 shape — one `query(X,who)` + one intransitive predicate over `X` + erasable event referent, nothing else; `what|which|how|howm|where|when`, restrictions, wh copula, multi-query = class `wh_query` errors. IR v2 gains a wh query constructor distinguishing the query variable (indicative: `query(query_id(...),wh(who),pred(Name,[var(1)]),source(...))`); still exactly one query per record; program goal carries the same form. Exact grammars are fixed in the owning unit's docs/ir.md rev (goal/answers shapes @ M4.1, IR constructor @ M4.2).
